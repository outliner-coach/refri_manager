냉장고 운영 에이전트 비개발자 가이드

버전: M1+M2 기준 | 업데이트: 2026-03-06

1. 이 시스템이 하는 일

이 시스템은 냉장고 음식 등록과 유통기한 알림을 자동화합니다. 사용자는 태블릿에서 사번 인증 후 음식 정보를 등록하고, 시스템은 유통기한이 다가오면 Slack으로 알려줍니다.

사번 인증 음성/사진 등록 유통기한 자동 알림 관리자 추적

2. 구조도 A: 전체 시스템

사람이 보는 화면과, 뒤에서 돌아가는 시스템 연결 구조

냉장고 앞 태블릿
(사번 + 음식 등록)
→
API 서버
(검증/저장/연동)
→
MySQL
(등록 데이터 저장)
관리자 웹
(현황 조회)
→
워커
(알림 스케줄 실행)
→
Slack
(본인/관리자 알림)
Google Sheets
(구성원 정보)
→
API 서버 동기화
(사번/이메일 매핑)
→
오브젝트 스토리지
(사진/음성 파일)

3. 구조도 B: 음식 등록 흐름

사용자 행동 → 시스템 처리 → 결과

1) 사번 입력
→
2) 구성원 확인
(Google Sheets 기반)
→
3) 음식명/유통기한/사진 등록
4) 유통기한 검증
(등록일 기준 6개월 이내)
→
5) DB 저장 + 알림 일정 생성
→
6) 등록 완료

4. 구조도 C: 알림 정책 타임라인

유통기한을 기준으로 알림이 자동 예약됩니다.

  • D-3: 본인 Slack DM 알림
  • D-day: 본인 Slack DM 알림
  • D+7: 본인 Slack DM + (미처리 시) 관리자 알림
  • D+14 이후: 미처리 상태이면 매주 본인 재알림

Slack 사용자 매핑이 실패하면 알림 누락을 막기 위해 관리자 채널로 fallback 알림이 갑니다.

5. 개인정보와 보관 정책

보관 기간

  • 이름/사진 로그 운영 조회: 1년
  • 데이터 삭제 기준: 2년 경과 후 자동 삭제

접근 원칙

  • 일반 사용자: 본인 항목만 수정 가능
  • 관리자: 전체 현황 확인
  • 모든 변경은 감사 로그에 기록

6. 운영자가 알아야 할 체크포인트

  • 정상: API Health OK, Worker processed 로그 증가
  • 주의: Slack 매핑 실패 fallback 발생 빈도 증가
  • 주의: 월 비용 80% 초과 경고
  • 조치: 구성원 이메일/사번 정보가 최신인지 Google Sheets 확인

7. 자주 묻는 질문 (운영 관점)

Q. 왜 사번 입력이 필요한가요?

A. 등록자 식별을 정확히 하고, 본인 알림을 보내기 위해서입니다.

Q. 유통기한을 너무 길게 넣으면 어떻게 되나요?

A. 등록 시 자동으로 거절됩니다. (등록일 기준 6개월 초과 불가)

Q. Slack 알림이 안 가면요?

A. 시스템이 재시도하고, 실패 시 관리자 채널로 자동 에스컬레이션합니다.
